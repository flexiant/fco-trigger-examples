Auto Start

Automatic starting of servers at deployment to save additional steps.

Key Type: Customer Key
Resource: Customer
Key Name: AUTO_START_SERVERS

Value: [Not Important]

 

Automatic Server Shutdown

Automatic sever shutdown based on the Customer key set on the server. Good for test/dev for internally facing services etc.

Key Type: Customer Key
Resource: Server
Key Name: SHUTDOWN_AFTER

Value: [Minutes after which server will shut down]

 

BentoBox Custom Startup Order

Enable servers within a bento box to be booted in a certain order for application compatability by pausing jobs to make them dependant on each other to boot in a certain order.

Key Type: Customer Key
Resource: Server (VM)
Key Name: START_SERVER

Value: [Number representing order in which servers will boot (e.g. 1, 2, 3, 4, 5, 6)]

You should set this key to all Servers (VM) in Bento box for this trigger to properly work.

Ducksboard Dashboard Management

Trigger that automatically posts to a Duckboard page with the number of servers running, number of customers, customer activations in the last 24 hours, invoices raised in the last 24 hours.

Key Type: Billing Entity Key

All URLs and API token are provided by Ducksboard

Resource: Billing Entity

Key Name: DUCKSBOARD_API_TOKEN

Value:
[Your API Token provided by Ducksboard]

Key Name: SERVER_DATA_URL

Value:

[URL to which Server Count should be sent]

Key Name: CUSTOMER_DATA_URL

Value:

[URL to which Customer Count should be sent]

Key Name: NEW_CUSTOMER_DATA_URL

Value:

[URL to which New Customer Count should be sent]

Key Name: NEW_INVOICE_DATA_URL

Value:

[URL to which New Invoices Count should be sent]

Firewall Deny All Rule

Automatically add deny all firewalls to each VM at create time, through creating a denyall firewall the first time, and adding additional servers to that existing template thereafter.

Key Type: Customer Key
Resource: Customer
Key Name: DENY_ALL_FIREWALL

Value: [Not Important]

Firewall used for this template is named DENY_ALL_FIREWALL_TEMPLATE and this name should be available for trigger to create a new firewall (if it doesnt already exist).

HipChat Messaging Integration

Send notifications of new customer activations, new server creations and new invoices created to a Hipchat channel.

Key Type: Customer Key

Notification Token doesnt seem to work.
Administrator token should be used instead.

Resource: Customer

Key Name: HIPCHAT_API_TOKEN

Value: [Your HipChat API Token (Administrator one)]

Key Name: HIPCHAT_ROOM_ID

Value: [ID of the room where notifications should be sent]

Loggly Log Managment

Automatic independent logging and analysis of important events for audit and stat analysis purposes.

Key Type: Billing Entity Key
Resource: Billing Entity
Key Name: LOGGLY_CUSTOMER_TOKEN

Value: [Your LOGGLY Customer API token]

 

MailChimp Integraion

Whenever a new customer is activated, or creates their first server, or their first bento box, this trigger will add them to a custom mailchimp mailing list. The mailing list details is configurable using Customer keys.

Key Type: Billing Entity Key

 

Resource: Billing Entity

Key Name: MAILCHIMP_TOKEN

Value: [Your MAILCHIMP API Token]

Key Name: MAILCHIMP_LIST_ID

Value: [List ID of the MailChimp Subscription List]

Scheduled Create Snapshot

Scheduled snapshots based on a customer system key to dictate the schedule.

Key Type: Customer Key
Resource: Server, Disk
Key Name: AUTO_SNAPSHOTS

Value: [Number representing how regularly snapshots will be taken (Hours)]

 

ServerDensity Monitoring

Automatically add servers to SeverDensity server monitoring system when sever is started for the first time.

Key Type: Customer Key
Resource: Customer
Key Name: SERVER_DENSITY_API_TOKEN

Value: [Your ServerDensity API token]

 

Signup-Login Email Alert

Sends out email whenever a new customer is created and/or whenever a customer logins to FCO.

Key Type: Billing Entity Key
Resource: Billing Entity
Key Name: SIGNUP_LOGIN_EMAIL

Value: [Email address to which notifications will be sent to]

 

Trial Customer Expiry

Automatic expiry of new user trial accounts and servers after certain time period if credit card not added.

Key Type: Billing Entity Key
Resource: Billing Entity
Key Name: TRIAL

Value: [Number representing how many days can customer be a trial]

Zapier Job Reports

Sends a list of all job notifications and their relevant data to a web hook for Zapier to be able to parse and manipulate the data.

Key Type: Billing Entity Key
Resource: Billing Entity
Key Name: ZAPIER_WEB_HOOK_URL

ZenDesk Error Job Reports

Each time a job fails in FCO, this trigger creates a ticket for it in ZenDesk.

Key Type: Billing Entity Key

 

Resource: Billing Entity

Key Name: ZENDESK_API_TOKEN

Value: [Your ZenDesk API Token]

Key Name: ZENDESK_DOMAIN_NAME

Value: [Subdomain Name registered at ZenDesk]
